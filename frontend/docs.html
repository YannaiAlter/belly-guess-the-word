Belly Jackpot Developer Documentation

Belly is now a timed jackpot game. Players join a round by paying the configured entry fee and receive a random ticket number. After the round ends, anyone can finalize and the contract picks one winner from all participants.

Back to app

Round Lifecycle

  1. Owner configures fee/duration/fee-bps while no round is active.
  2. Owner starts a new round with startNewRound().
  3. Users call joinJackpot() with exact entry fee.
  4. After roundEndsAt, anyone calls finalizeJackpot().
  5. Contract computes winner index using on-chain entropy and pays winner + owner fee.

Write Methods

  • startNewRound() — owner only.
  • joinJackpot() — payable; exact entryFeeWei.
  • finalizeJackpot() — callable by anyone after deadline.
  • setEntryFee(uint256) — owner only, inactive round only.
  • setRoundDuration(uint256) — owner only, inactive round only.
  • setFeeBps(uint16) — owner only, inactive round only.

Read Methods

  • getRoundState() — full round snapshot for frontend.
  • getMyTicket(roundId, address) — participant ticket lookup.
  • getParticipantsCount() and getParticipantAt(index).
  • timeRemaining() — seconds left until finalization window.

Events

  • RoundStarted
  • ParticipantJoined
  • RoundFinalized
  • EntryFeeUpdated
  • RoundDurationUpdated
  • FeeBpsUpdated

Frontend Contract API

Deployment writes frontend/contract.json with address + ABI. The frontend loads this file and instantiates ethers contract:

const res = await fetch("./contract.json");
const info = await res.json();
const contract = new ethers.Contract(info.address, info.abi, signer);

Randomness Caveat

The winner is selected using pseudo-random on-chain entropy (block.prevrandao and round data). This is practical and dependency-free, but not equivalent to a VRF oracle for adversarial high-value games.
